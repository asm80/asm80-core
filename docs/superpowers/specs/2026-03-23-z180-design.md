# Z180 Assembler Support — Design Spec

**Date:** 2026-03-23
**Status:** Approved

---

## Goal

Add Zilog Z180 / Hitachi HD64180 as a new CPU target in asm80-core. The Z180 is a strict superset of Z80: every valid Z80 instruction is valid Z180, but Z180 defines 10 additional mnemonics (covering 34 opcode slots) using ED-prefixed opcodes that are undefined/illegal on plain Z80.

---

## Architecture: Wrapper Pattern

`cpu/z180.js` follows the same pattern as `cpu/h6309.js` (HD6309 extends M6809):

```js
import { Z80 } from "./z80.js";
export const Z180 = { ...Z80, cpu: "z180", ext: "z180", set: {...Z80.set, ...z180Extensions}, parseOpcode };
```

**Why wrapper, not standalone:** duplicating Z80's ~300-line opcode table and parseOpcode would create a maintenance burden. Any future Z80 fix would need to be applied twice.

### Key architectural insight

`Z80.parseOpcode` hardcodes references to `Z80.set` and `Z80.R8` internally. Delegation therefore works correctly for all Z80 instructions. Z180-specific instructions must be intercepted **before** delegating to `Z80.parseOpcode`, because they do not exist in `Z80.set`.

The `set` property on the CPU object is **not read by the assembler core** (`parseLine.js` only calls `parseOpcode`). The merged `Z180.set` exists for external tooling (Monaco editor autocomplete/syntax highlighting).

## New Instructions

All new instructions use the ED prefix. Encoding is computed arithmetically in `parseOpcode` special cases — not via table dispatch.

### TST — Test (AND with A, flags only, no store)

| Form | Bytes | Encoding |
|------|-------|----------|
| TST r | 2 | ED, 04 + (regIndex * 8) |
| TST (HL) | 2 | ED 34 |
| TST n | 3 | ED 64 n |

Register order for index: B=0, C=1, D=2, E=3, H=4, L=5, A=7 (index 6 = (HL) slot, not used here).

### TSTIO n — Test I/O (AND (C) with n, flags only)

| Form | Bytes | Encoding |
|------|-------|----------|
| TSTIO n | 3 | ED 74 n |

### MLT rr — 8×8 Multiply

| Form | Bytes | Encoding |
|------|-------|----------|
| MLT rr | 2 | ED, 4C + (pairIndex * 16) |

Pair order: BC=0, DE=1, HL=2, SP=3.

### SLP — Sleep

| Form | Bytes | Encoding |
|------|-------|----------|
| SLP | 2 | ED 76 |

### Block output instructions (no operands)

| Mnemonic | Bytes | Encoding |
|----------|-------|----------|
| OTIM | 2 | ED 83 |
| OTDM | 2 | ED 8B |
| OTIMR | 2 | ED 93 |
| OTDMR | 2 | ED 9B |

### IN0 r,(n) — Input from absolute port

| Form | Bytes | Encoding |
|------|-------|----------|
| IN0 r,(n) | 3 | ED, (regIndex * 8), n |

Register order: B=0, C=1, D=2, E=3, H=4, L=5, A=7. Index 6 is invalid (no (HL) form).
`n` is an 8-bit immediate port address, expressed as `(expr)` — parentheses are stripped by `parseOpcode`.

### OUT0 (n),r — Output to absolute port

| Form | Bytes | Encoding |
|------|-------|----------|
| OUT0 (n),r | 3 | ED, (regIndex * 8) + 1, n |

Same register order as IN0.

## parseOpcode Design

```
Z180.parseOpcode(s, vars, Parser):
  if TST:
    if param is (HL)  → [ED, 34]
    if param is r8    → [ED, 04 + ri*8]
    else (immediate)  → [ED, 64, n]
  if TSTIO  → [ED, 74, n]
  if MLT    → [ED, 4C + ri*16]
  if SLP/OTIM/OTDM/OTIMR/OTDMR → [hi, lo] from fixed table
  if IN0    → [ED, ri*8, port]
  if OUT0   → [ED, ri*8+1, port]
  else → Z80.parseOpcode(s, vars, Parser)
```

### Edge case: TST (expr) vs TST (HL)

`TST (HL)` is detected by exact string match `"(HL)"`. Any other parenthesised expression (e.g. `TST (MY_ADDR)`) falls through to the immediate branch, which will evaluate the expression — this produces wrong semantics but is not a defined Z180 instruction anyway. Acceptable to let it assemble or throw.

## Registration

In `asm.js`:

```js
import { Z180 } from "./cpu/z180.js";
// add to cpus array:
const cpus = [..., Z180];
```

CPU name `"z180"` and alias `"hd64180"` are both resolved via `.cpu` directive matching `Z180.cpu`. If an alias is needed, a second object `{ ...Z180, cpu: "hd64180" }` can be added to the array.

## Tests

New file `test/asm-z180.js`. Pattern follows `test/asm-6309.js` — direct unit tests of `Z180.parseOpcode`.

### Selective test execution

Add to `package.json` scripts:

```json
"test:z180": "qunit test/asm-z180.js",
"test:z80":  "qunit test/asm-z80.js",
"test:fast": "qunit"
```

`test:fast` runs all tests without c8 coverage (faster during development).

## What this does NOT change

- Z80 instruction encoding — untouched, delegated as-is
- Undocumented Z80 instructions (IXH, IXL, etc.) — Z180 treats them as illegal; the Z80 encoder already omits them
- Relocatable module format, linker, directives — unchanged
- Address space model — 16-bit, 64 KB, identical to Z80
